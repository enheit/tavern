# S3.3 manual QA ‚Äî restart stays logged in (macOS)

The automated DoD gates pass without a device:

- **cargo** (`cargo test -p tavern-desktop`): `CredentialStore` roundtrip through the
  in-memory mock (save‚Üíload‚Üíclear) + `Session` camelCase serde guard.
- **mockIPC** (`app/src/lib/boot.test.ts`): valid stored token ‚Üí session active +
  servers loaded + `engine_configure` invoked with `{apiBase, token}`; invalid token ‚Üí
  `session_clear` invoked ‚Üí stays on Onboarding.

This file covers the one thing those cannot: the **real OS keychain surviving a full
process restart**. Run it on the dev machine.

## Procedure

1. `pnpm dev:worker` (local Miniflare: D1/DO/R2 on disk ‚Äî sessions persist across app
   restarts because the token lives in D1 + the macOS login keychain).
2. In another shell: `pnpm tauri dev`.
3. In the app: **Create account** (e.g. `qauser` / `hunter2pw`). Land on the Main shell.
   - macOS shows a keychain prompt the first time the app writes the session ‚Äî choose
     **Always Allow** for `tavern-desktop`.
4. Fully **quit** the app (Cmd-Q / close the dev window; stop `pnpm tauri dev`).
5. Relaunch: `pnpm tauri dev`.

## Expected

- On relaunch the app boots straight to the **Main shell** (not Onboarding) ‚Äî boot read
  `{userId, token}` from the keychain, `GET /api/me` returned 200, and
  `engine_configure` fired.
- `security find-generic-password -s app.tavern.desktop -a session` lists the item (the
  stored session JSON ‚Äî do not print its value into the repo).
- Logout (S3.4) then relaunch ‚Üí boots to **Onboarding** (keychain item cleared).
